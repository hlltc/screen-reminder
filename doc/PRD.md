## 1. 产品概述

### 1.1 产品定位

**Screen Reminder** 是一款面向长时间使用电脑的知识工作者（以程序员为核心用户群）的桌面端健康管理工具。产品通过**智能感知 + 渐进式提醒 + 行为数据追踪 + 游戏化激励**，帮助用户建立健康的屏幕使用习惯，预防和减轻数字眼疲劳（CVS）、颈椎/腰椎劳损、久坐综合症等职业健康问题。

### 1.2 核心设计理念

| 原则 | 说明 |
|------|------|
| **非侵入式（Non-intrusive）** | 提醒优先使用温和的视觉/触觉提示，避免打断心流状态 |
| **智能感知（Context-aware）** | 基于输入活动、摄像头姿态检测、日历事件等上下文动态调整提醒策略 |
| **渐进增强（Progressive）** | 提醒力度由浅入深：边缘浮动提示 → 半透明遮罩 → 全屏锁定，给用户选择余地 |
| **数据驱动（Data-driven）** | 记录并可视化健康行为数据，让改善可量化、可追踪 |
| **隐私优先（Privacy-first）** | 所有敏感数据（摄像头画面、生物特征）本地处理，不上传云端 |

### 1.3 产品形态

- **主形态**：跨平台桌面应用（Windows / macOS / Linux），以**系统托盘小组件**为常驻入口
- **运行方式**：开机自启 → 最小化到系统托盘 → 后台静默运行 → 按需唤出
- **扩展方向**：移动端伴侣 App（查看报告）、Apple Watch / Wear OS 手表端提醒、浏览器扩展联动

## 2. 市场背景与竞品分析

### 2.1 市场数据与痛点验证

| 痛点 | 数据支撑 | 来源 |
|------|----------|------|
| **数字眼疲劳（CVS）** | 全球程序员群体 CVS 患病率 **55%–77%**；中国程序员近视率超 90% | 2025-2026 多国横断面研究 |
| **颈椎问题** | 头部前倾 30° 时颈椎承压达 **18kg**（正常 5kg）；程序员颈椎病发病率 > 50% | 临床生物力学研究 |
| **腰椎问题** | 程序员腰椎发病率超 **60%**；连续 2 小时不当坐姿使椎间盘压力增加 **40%** | 职业健康流行病学调查 |
| **久坐危害** | 每天久坐 > 8 小时全因死亡率增加 **59%**；每静坐 30 分钟打断一次可显著降低代谢风险 | 柳叶刀 / WHO 指南 |
| **水分摄入不足** | **75%** 职场人存在慢性轻度脱水，认知能力下降 10–20% | 营养学 Meta 分析 |

### 2.3 市场空白与差异化机会

| 竞品短板 | Screen Reminder 差异化 |
|----------|------------------------|
| 多数产品仅覆盖 1–2 个健康维度 | **全维度覆盖**：眼、颈、腰、久坐、补水、呼吸六合一 |
| 提醒策略粗暴，打断工作流 | **渐进式提醒 + 心流感知**，先轻后重 |
| 缺乏输入行为感知，定时触发 | **APM 感知 + 空闲检测**，动态调整间隔 |
| 无姿态检测或需云处理 | **本地姿态检测**（摄像头），隐私安全 |
| 数据孤立，无洞察 | **健康仪表盘 + 趋势分析 + 导出报告** |
| 缺乏激励机制 | **成就系统 + 连续打卡 + 健康等级** |

## 3. 目标用户与用户画像

### 3.1 核心用户：程序员

```
姓名：小张
年龄：28
职业：后端开发工程师
日均屏幕时间：10–12 小时
健康困扰：
  - 干眼、视力模糊，下午头痛频发
  - 颈椎僵硬，每周需要按摩缓解
  - 久坐下背酸痛，早晨起床弯腰困难
  - 常常忘记喝水，日均饮水 < 1L
  - 代码写到入迷一坐 3 小时不动
需求：
  - 希望有非打扰式的提醒，不打断编码心流
  - 需要量化健康数据，看到改善效果
  - 喜欢游戏化机制，愿意为健康成就"打卡"
```

### 3.2 扩展用户

- **远程办公者**：居家办公缺乏办公室走动契机，自律性要求更高
- **设计师 / 视频剪辑师**：同样长时间盯屏，手部重复性劳损风险
- **大学生 / 研究生**：学术写作、论文阅读，CVS 高风险人群（患病率 > 73%）
- **金融 / 数据分析从业者**：多屏工作，注意力高度集中

## 4. 核心痛点与产品价值主张

### 4.1 痛点—解决方案映射

| 用户痛点 | 当前行为 | 产品解决方案 |
|----------|----------|-------------|
| 忘记休息 | 意识到时已连续工作 3h+ | 智能感知工作状态，20-20-20 护眼提醒 + 55min 久坐提醒 |
| 坐姿变形 | 无意识地头前倾、驼背 | 摄像头实时姿态检测，前倾 >15° 触觉震动提醒 |
| 忘记喝水 | 下班才发现水瓶没开 | 定时补水提醒 + 一键记录 + 每日水量目标追踪 |
| 颈椎僵硬 | 只有疼痛时才活动 | 定时颈部拉伸引导 + 3D 动画演示 |
| 缺乏动力 | 健康习惯难以坚持 | 连续打卡、成就徽章、健康评分、趋势可视化 |
| 提醒打断工作 | 关掉闹钟后继续干活 | 渐进式提醒：边缘闪烁 → 半透明遮罩 → 全屏锁定 |
| 会议室 / 桌面模式不同 | 手动暂停/恢复 | 摄像头人形检测 + 日程感知，自动识别离开/会议状态 |

### 4.2 一句话价值主张

> **一个不打断你编码的桌面健康管家，默默守护你的眼睛、颈椎、腰椎。**

### 6.1 护眼模块（Eye Care）

#### 6.1.1 20-20-20 法则提醒

| 属性 | 描述 |
|------|------|
| **触发条件** | 屏幕活动连续达到 20 分钟 |
| **提醒内容** | "请看向 6 米（20 英尺）外的物体，持续 20 秒" |
| **提醒方式** | 阶段 1：屏幕右下角浮动提示（5s）→ 阶段 2：屏幕边缘半透明绿色遮罩（15s 未响应）→ 阶段 3：全屏 20s 倒计时放松画面 |
| **倒计时** | 阶段 3 显示 20s 倒计时 + 远处景观图片/动画，引导用户望向远处 |
| **智能延迟** | 若摄像头检测到用户不在屏幕前（起身、离开），自动重置倒计时 |
| **可配置项** | 间隔（15/20/25/30 min），休息时长（15/20/30/60 s），提醒强度（仅提示/温和/严格） |

#### 6.1.2 蓝光过滤

| 属性 | 描述 |
|------|------|
| **自动调度** | 根据地理位置日出日落时间自动开关 |
| **色温范围** | 2700K（暖黄）– 6500K（标准白）可平滑过渡 |
| **强度可调** | 滑块控制过滤强度 0–100% |
| **手动模式** | 支持始终开启 / 始终关闭 / 定时 / 日出日落自动 |

#### 6.1.3 环境光自适应

| 属性 | 描述 |
|------|------|
| **传感器** | 调用笔记本光线传感器（如有）或摄像头估算环境光 |
| **自适应策略** | 环境光 < 200 lux → 提示开灯 / 降低蓝光；环境光 > 1000 lux → 提示防眩光 |
| **屏幕亮度联动** | 可选自动调节屏幕亮度匹配环境光 |

#### 6.1.4 用眼报告

| 属性 | 描述 |
|------|------|
| **日报告** | 当日屏幕总时长、有效休息次数、20-20-20 完成率、连续用眼最长时段 |
| **周报告** | 环比趋势图、日均用眼时长、休息完成率变化 |
| **健康评分** | 综合用眼时长、休息频率、环境光质量打 0–100 分 |

### 6.2 颈椎健康模块（Neck Care）

#### 6.2.1 姿态检测提醒

| 属性 | 描述 |
|------|------|
| **检测方式** | 摄像头（本地 MediaPipe/OpenCV 17 点人体关键点追踪） |
| **监测指标** | ① 头前倾角度（耳垂—肩峰连线偏离垂直线角度）；② 高低肩；③ 颈部侧倾 |
| **触发阈值** | 前倾 > 15° 持续 > 30s → 触觉震动/图标变色提醒 |
| **提醒方式** | 任务栏图标绿（正常）→ 黄（注意，15°–25°）→ 红（警告，> 25°）；可选桌面通知 "坐直一点哦" |
| **隐私保障** | 所有视频帧在内存中处理，不存储、不上传，软件图标显示摄像头工作状态 |

#### 6.2.2 颈部拉伸引导

| 属性 | 描述 |
|------|------|
| **触发** | ① 每 60 分钟主动提醒一次颈部拉伸；② 姿态检测连续不良 > 5 分钟触发 |
| **内容** | 3D 动画 / GIF 引导：左右侧屈、旋转、前后活动，每个动作 15s × 4 组 |
| **总时长** | 约 1–2 分钟 |
| **跳过** | 允许跳过（记录跳过次数用于健康报告） |

#### 6.2.3 颈椎健康报告

| 属性 | 描述 |
|------|------|
| **指标** | 日均前倾时长占比、良好姿态率、拉伸完成率、疼痛自评趋势 |
| **趋势** | 周/月对比，判断改善或恶化 |

### 6.3 腰椎健康模块（Spine Care）

#### 6.3.1 久坐提醒

| 属性 | 描述 |
|------|------|
| **默认间隔** | 连续坐姿 55 分钟后提醒 |
| **强制级别** | 级 1：桌面通知 "该站起来了"；级 2（2 分钟未响应）：半透明遮罩 "站起来活动 3 分钟" |
| **站立倒计时** | 遮罩显示 3 分钟倒计时，倒计时期间键盘鼠标输入受限（可紧急跳过） |
| **可配置** | 间隔（30/45/55/60 min），站立时长（1/2/3/5 min），严格程度（仅提醒/软锁定/硬锁定） |

#### 6.3.2 坐站交替建议

| 属性 | 描述 |
|------|------|
| **统计** | 记录每日坐姿时间和站立时间 |
| **建议** | 推荐 1:1 坐站比，显示当前比值和理想比值 |
| **适用** | 拥有升降桌的用户（可选择开启/关闭此功能模式） |

#### 6.3.3 腰部拉伸引导

| 属性 | 描述 |
|------|------|
| **触发** | ① 久坐提醒后自动进入；② 可手动触发 |
| **内容** | 坐姿扭转、前屈、侧屈、猫牛式等 6 组动作，3D 动画指导 |
| **时长** | 约 2–3 分钟 |

### 6.4 站立与走动模块（Movement）

#### 6.4.1 走动提醒

| 属性 | 描述 |
|------|------|
| **触发** | 每 90 分钟（可配置）提醒起身走动 |
| **提醒内容** | "走几步休息一下吧！接杯水、看看窗外" |
| **目标** | 每日步行中断 ≥ 6 次，每次 ≥ 2 分钟 |

#### 6.4.2 伸懒腰提醒

| 属性 | 描述 |
|------|------|
| **触发** | 每 30 分钟（独立于其他提醒） |
| **内容** | 简短提示 + 动画：双臂上举、后仰伸展 |
| **特征** | 极轻量提醒（仅工具栏闪烁），5 秒内完成，几乎不打断工作 |

#### 6.4.3 计步联动（可选）

| 属性 | 描述 |
|------|------|
| **数据源** | 手机/手表计步器（通过蓝牙或局域网同步） |
| **目标** | 设置每日步数目标（默认 6000 步），未达标时下午加强走动提醒 |

### 6.5 补水提醒模块（Hydration）

#### 6.5.1 定时补水提醒

| 属性 | 描述 |
|------|------|
| **默认间隔** | 每 45 分钟提醒一次 |
| **提醒方式** | 桌面轻通知："该喝水了！💧" |
| **快捷操作** | 通知内嵌按钮：「已喝 200ml」「已喝 300ml」「跳过」 |
| **可配置** | 间隔（30/45/60 min），单次饮水量（150/200/250/300ml），提醒时段（工作时间） |

#### 6.5.2 每日水量目标

| 属性 | 描述 |
|------|------|
| **默认目标** | 2000ml（可根据体重自动计算：体重 × 30ml） |
| **进度条** | 系统托盘图标显示水滴进度；桌面悬浮小窗显示当日进度环 |
| **达成提醒** | 达到目标后推送庆祝通知 |

#### 6.5.3 水温建议（可选）

| 属性 | 描述 |
|------|------|
| **策略** | 根据时间段推荐：上午温水、下午常温水、晚间少饮 |
| **联动** | 可与智能水杯硬件对接（远期规划） |

### 6.6 微运动与拉伸模块（Stretching）

#### 6.6.1 动作库

| 分类 | 动作列表 | 适用场景 |
|------|----------|----------|
| **颈部** | 左右侧屈、前后活动、颈部旋转 | 颈部僵硬 |
| **肩部** | 耸肩、肩关节环绕、背后扣手 | 圆肩含胸 |
| **背部** | 猫牛式、坐姿扭转、侧屈 | 久坐下背痛 |
| **手腕** | 手腕旋转、抗阻伸展、手指抓握 | 键盘手 |
| **眼部** | 远近焦点切换、眼球转动、热掌敷眼 | 眼疲劳 |

#### 6.6.2 展示方式

- **3D 骨骼动画**（使用 Lottie / Rive / 自定义渲染）

### 6.8 智能调度引擎（Smart Scheduler）

这是产品的**核心差异化模块**，负责统一管理所有提醒的触发时机和强度。

#### 6.8.1 工作强度感知

| 指标 | 来源 | 用途 |
|------|------|------|
| **APM（每分钟操作次数）** | 键盘/鼠标 hook | APM > 200 → 高强度编码，延迟低优先级提醒 |
| **鼠标移动距离** | 鼠标 hook | 持续小范围移动 → 精细操作（设计/编码），暂停打断 |
| **连续活跃时长** | 系统空闲检测 | 累计有效活跃时间，非单纯挂机 |
| **应用程序类型** | 前台窗口检测 | IDE/编辑器 → 编码模式；浏览器 → 浏览模式；终端 → 运维模式 |

#### 6.8.2 日历联动

| 功能 | 描述 |
|------|------|
| **会议检测** | 读取系统日历（Outlook / Google Calendar），会议期间自动暂停全屏遮罩类提醒 |
| **专注时段** | 用户标记"请勿打扰"时段，只保留最轻量提醒 |
| **休息空档** | 自动识别日程空档（> 5 min），推荐拉伸/走动 |

#### 6.8.3 空闲检测

| 功能 | 描述 |
|------|------|
| **离开检测** | 鼠标键盘无操作 > 2 分钟 → 认为用户已离开，重置所有计时器 |
| **摄像头辅助** | 人形检测确认用户是否在屏幕前，减少误判 |
| **睡眠检测** | OS 睡眠/休眠事件监听，唤醒后自动恢复计时 |

#### 6.8.4 提醒优先级与防抖

```
优先级层级：
  L1（紧急）: 久坐锁定（55min 无中断）、严重姿态异常（前倾 > 30° 持续 3min）
  L2（重要）: 20-20-20 护眼、补水提醒、颈部拉伸
  L3（建议）: 伸懒腰、走动建议、呼吸练习

防抖逻辑：
  - 短时间内（2 min）不重复发送同类型提醒
  - 高优先级提醒可覆盖低优先级提醒
  - 用户刚完成一项休息后，所有计时器重置
```

### 7.1 技术选型总览

| 维度 | 主选方案 | 备选方案 | 选择理由 |
|------|----------|----------|----------|
| **编程语言** | **Python 3.12+** | Node.js (Electron) | (见下方选型论证) |
| **GUI 框架** | **PySide6** (Qt for Python) | Flet / Tkinter | 原生级跨平台 UI、成熟系统托盘 API、硬件加速渲染 |

| **系统托盘** | **pystray** + PySide6 托盘 | rumps (macOS) | 跨平台、支持菜单/图标动态更新/Lottie |
| **日程读取** | **caldav** / Google API | 各平台原生 | 跨平台统一 API，支持 Outlook Calendar / Google Calendar |

| **动画资源** | **Lottie** (JSON 动画) / SVGs | QML 动画 | 跨平台轻量动画，拉伸动作展示 |

### 7.2 为什么选 Python 而非 Electron？

| 对比维度 | Python (PySide6) | Electron (Node.js) | 结论 |
|----------|------------------|---------------------|------|
| **内存占用** | 80–120 MB（空闲） | 200–400 MB（空壳即 250MB+） | Python 胜（常驻应用内存敏感） |
| **CPU 占用** | < 2% 后台 | 3–8%（Chromium 渲染） | Python 胜 |
| **AI/ML 生态** | MediaPipe、OpenCV、ONNX 第一方支持 | 需通过 WASM/子进程桥接 | Python 胜 |
| **系统 API 调用** | ctypes / PySide6 原生 | 需 native addon (C++) | 打平 |
| **UI 开发效率** | Qt Designer 快速搭建 | Web 技术，前端生态极强 | Electron 胜 |
| **打包体积** | ~60–80 MB (PyInstaller) | ~120–180 MB (electron-builder) | Python 胜 |
| **跨平台一致性** | Qt 跨平台成熟（30 年） | Chromium 天然一致 | 打平 |
| **社区 & 文档** | Qt 30 年 / Python 全球第二大语言 | 全球最大 Web 生态 | 打平 |

> **结论**：对于一款**始终在后台运行的桌面健康管家**，Python + PySide6 在资源占用、ML 生态、系统集成三个关键维度上均优于 Electron。Electron 的核心优势（丰富的 Web 前端生态）在此类应用场景中不是刚需。

### 7.7 打包与分发策略

| 平台 | 打包工具 | 输出格式 | 备注 |
|------|----------|----------|------|
| **Windows** | Nuitka + Inno Setup | `.exe` 安装包 | 编译为 C 可提升 2-3x 启动速度 |
| **macOS** | PyInstaller + DMG Canvas | `.dmg` | 需 Apple 开发者签名公证 |
| **Linux** | PyInstaller | `.AppImage` / `.deb` / Flatpak | 推荐 Flatpak 统一分发 |

### 7.8 性能约束

| 指标 | 目标值 | 备注 |
|------|--------|------|
| 后台 CPU 占用 | < 2%（不含推理） | MediaPipe 推理时 < 15% |
| 内存占用（空闲） | < 120 MB | Python 进程 + Qt 框架 |
| 内存占用（推理中） | < 250 MB | 摄像头 + MediaPipe 模型加载 |
| 姿态推理帧率 | 3–5 FPS | 仅采样级，非连续视频流 |
| 启动时间 | < 3 秒 | 冷启动（不含首次模型加载） |
| 打包体积 | < 80 MB | PyInstaller onefile 压缩 |

### 7.9 隐私与安全

- **本地处理**：摄像头画面仅用于本地 MediaPipe 推理，逐帧处理后立即释放内存，不存储、不上传
- **数据加密**：SQLite 数据库文件使用 pysqlcipher3（AES-256）加密
- **透明告知**：系统托盘图标实时显示摄像头工作状态（绿色圆点=检测中）
- **权限最小化**：摄像头权限仅在用户开启姿态检测功能时请求，可随时关闭模块
- **网络隔离**：应用无需网络权限即可正常运行（仅日历同步/更新检查可选联网）

## 8. UI/UX 设计规范

### 8.1 核心交互原则

1. **尽量少打断**：85% 的提醒通过托盘图标变化、桌面角落 Toast 完成；仅 15% 使用遮罩/锁定
2. **操作零成本**：所有提醒提供一键操作（"已喝水"、"跳过"、"5 分钟后提醒"）
3. **视觉舒适**：遮罩使用半透明深色 + 模糊背景，保护眼睛不刺眼
4. **动画自然**：使用缓动曲线，避免突兀的出现/消失
5. **信息降噪**：主界面仅展示今日最关键的 4–6 个指标，深层数据按需展开

### 8.2 设计风格

| 维度 | 描述 |
|------|------|
| **风格** | 现代简约 + 轻微玻璃拟态（Fluent Design / Acrylic） |
| **配色** | 主色调柔和蓝绿色（#4ECDC4），辅助色暖橙（#FF6B6B）用于警告 |
| **圆角** | 8–12px |
| **字体** | 中文：微软雅黑 / PingFang SC / Noto Sans CJK；英文：Segoe UI / SF Pro / Inter |
| **图标** | 线性图标为主，健康维度图标可着色表示状态 |

### 8.3 窗口形态

| 窗口类型 | 尺寸 | 特征 |
|----------|------|------|
| **主面板** | 400×600（可缩放） | 可吸附屏幕边缘自动隐藏 |
| **休息遮罩** | 全屏 | 半透明背景 + 中央内容卡片 |
| **迷你窗口** | 200×200 | 悬浮圆环进度，始终置顶（可选开启） |
| **系统托盘弹出小组件** | 260×380 | 轻量级卡片，点击托盘图标弹出（见 8.4 详述） |

### 8.4 系统托盘小组件设计（核心入口）

这是产品的**主要日常交互界面**。用户 90% 的操作在此完成，无需打开主面板。

#### 8.4.1 托盘图标状态

```
🟢 正常待机      — 所有健康指标正常
🟡 需注意        — 某项指标接近阈值（如用眼快满 20 分钟）
🔴 需行动        — 有提醒未响应（久坐/饮水/姿态异常）
⏸️ 暂停中        — 用户手动暂停（午休 / 会议中）
```

图标设计为双色渐变圆形，中心叠加当前最紧急的健康符号（眼/水滴/人形）。

#### 8.4.2 弹出小组件布局（Tray Popup Card）

```
┌──────────────────────────────────────┐
│  🟢 下午好，小张          2026-05-23 │ ← 问候栏
├──────────────────────────────────────┤
│                                      │
│     ◉ ◉ ◉                            │
│   护眼  补水  活动                    │ ← 三色健康圆环
│   ╱    ╱    ╱                        │   (类似 Apple Watch Rings)
│  85%  60%  70%                       │
│                                      │
├──────────────────────────────────────┤
│  ⏱ 距离下次休息    12 分钟            │ ← 倒计时行
│  💧 已喝 800 / 2000ml              │ ← 补水进度
│  🧘 坐姿状态        良好            │ ← 姿态指示
├──────────────────────────────────────┤
│  [ ☕ 5分钟休息 ]                     │ ← 快捷操作按钮
│  [ 🧘 颈部拉伸 ]  [ 💧 喝了]         │
├──────────────────────────────────────┤
│  🏆 连续达标 7 天  │ 📊 详细报告 → │ ← 成就 + 跳转
└──────────────────────────────────────┘
```

#### 8.4.3 右键托盘菜单


#### 8.4.4 组件交互规范

| 交互 | 行为 |
|------|------|
| **左键单击托盘图标** | 弹出小组件卡片（show/hide toggle） |

## 10. 成功指标

### 10.1 北极星指标

> **每周活跃用户的日均健康行为完成率** = (实际完成休息次数 / 应完成休息次数) × 100%

### 10.2 关键结果（KRs）

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 日活用户 20-20-20 完成率 | > 60% | 应用内统计 |
| 日均有效休息次数 | ≥ 8 次 | 应用内统计 |
| 日均补水达标率 | > 70% | 用户记录数据 |
| 用户连续使用天数 | 中位数 ≥ 14 天 | 应用内统计 |
| 用户自评健康改善 | > 70% 报告有改善 | 内置 NPS 问卷（30 天触发） |
| 后台 CPU 占用 | < 2% | 性能监控 |
| 崩溃率 | < 0.5% | 崩溃收集 |

### 10.3 用户反馈指标

| 反馈维度 | 收集方式 |
|----------|----------|
| NPS（净推荐值） | 使用 30 天后弹出 NPS 打分 |
| 功能满意度 | 模块使用率 + "不再提醒"关闭率 |
| 痛点反馈 | 应用内反馈入口 + 邮件/Issue |
